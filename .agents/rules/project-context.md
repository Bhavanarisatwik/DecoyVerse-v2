 ## Project Context: DecoyVerse

  DecoyVerse is a cybersecurity deception platform. It plants honeytoken files on monitored endpoints and uses ML to detect and score attackers who access them.

  ---

  ### Monorepo Structure

  | Directory | Stack | Purpose |
  |-----------|-------|---------|
  | `DecoyVerse-v2/` | React 19 + Vite + TypeScript + Tailwind CSS v4 + Express + MongoDB | Frontend SPA + auth/email backend |
  | `ML-modle v0/` | Python + FastAPI | Data backend + ML prediction service + endpoint agent |
  | `Proforpro/` | Astro | Documentation site |

  ---

  ### Architecture

  Frontend React (:5173)
    ├── authClient → Express (:5000)    # Auth, profile, vault, email
    └── apiClient  → FastAPI (:8001)    # Nodes, decoys, alerts, logs, AI insights
                         └── ML API (:8000)    # Attack prediction (internal only, never called from frontend)

  Endpoint Agent (agent.py, deployed on monitored machines)
    └── X-Node-Id + X-Node-Key → POST /api/agent-alert → FastAPI → ML score → alert if score ≥ 7

  ### Auth

  - JWT stored in `localStorage` as `token`
  - Both `authClient` and `apiClient` auto-inject `Authorization: Bearer <token>`
  - 401 → clear storage + redirect to `/auth/login`
  - `JWT_SECRET` (Express `.env`) **must equal** `JWT_SECRET_KEY` (FastAPI `.env`)
  - Agent auth uses `X-Node-Id` + `X-Node-Key` headers (bcrypt-validated, not JWT)
  - `INTERNAL_SECRET` must match on both Express and FastAPI (used for Python→Express email relay)

  ---

  ### Email

  Email is **never sent by Python directly** — Render blocks SMTP ports 25/465/587.
  Flow: `FastAPI notification_service.py` → HTTP POST to `Express /api/auth/internal/send-alert-email` (with `x-internal-secret` header) → Express sends via SendGrid HTTPS API.

  ### Frontend Key Files

  - `src/api/client.ts` — dual Axios instances (`authClient` → Express, `apiClient` → FastAPI)
  - `src/api/types.ts` — all TypeScript interfaces
  - `src/api/mockData.ts` — demo mode data (**must stay in sync with real API shapes**)
  - `src/context/AuthContext.tsx` — JWT auth state, validates on mount via `GET /api/auth/me`
  - `src/context/ThemeContext.tsx` — `gold`/`orange`/`light`, stored as `decoyverse-theme` in localStorage
  - `src/App.tsx` — all routes (public, protected, onboarding)
  - `src/utils/cn.ts` — **always use `cn()` for className merging, never string concatenation**

  ### Frontend Conventions

  - Accent colors: `gold-400/500/600`. Cards: `bg-gray-800 border-gray-700 rounded-xl`
  - Status badges: `status-success | status-info | status-warning | status-danger`
  - Alert status values: `'open' | 'acknowledged' | 'investigating' | 'resolved'` — **never `'new'`**
  - TypeScript: use `interface` for shapes, avoid `any`. Pages: default exports. Components: named exports.
  - New page checklist: `src/pages/X.tsx` → route in `App.tsx` → nav in `src/components/layout/Sidebar.tsx`
  - New Express endpoint: `server/src/routes/x.ts` → mount in `server/src/index.ts` → `src/api/endpoints/x.ts` → re-export from `src/api/index.ts`
  - New FastAPI endpoint: `ML-modle v0/backend/routes/x.py` → register in `ML-modle v0/backend/main.py` → `src/api/endpoints/x.ts` → re-export from `src/api/index.ts`

  ---

  ### FastAPI / ML Key Files

  - `backend/models/log_models.py` — **all Pydantic models live here** (edit first for any schema change)
  - `backend/services/db_service.py` — **all MongoDB operations** (never call pymongo directly in routes)
  - `backend/services/ml_service.py` — ML API integration + `_convert_to_ml_features()` heuristics
  - `backend/services/notification_service.py` — Slack, Email relay, WhatsApp (always use this, never call directly from routes)
  - `backend/config.py` — all config including `ALERT_RISK_THRESHOLD = 7`

  ### ML Pipeline

  6 input features (fixed order — never change):
  `failed_logins, request_rate, commands_count, sql_payload, honeytoken_access, session_time`

  Attack types: `Normal | BruteForce | Injection | DataExfil | Recon`
  Risk score 1–10. Alert threshold: **≥ 7** triggers alert + notifications.
  Models are `.pkl` files generated by `train_model.py`.

  ### Data Scoping

  **Every MongoDB query must include `user_id` filter** — all data is user-scoped. Missing this leaks cross-user data.

  ---

  ### Cross-Service Change Order

  When adding a new field to any shared data model (e.g. alert, node, event):
  1. `ML-modle v0/backend/models/log_models.py` — Pydantic model
  2. `ML-modle v0/backend/services/db_service.py` — MongoDB read/write
  3. `DecoyVerse-v2/src/api/types.ts` — TypeScript interface
  4. `DecoyVerse-v2/src/api/mockData.ts` — demo mock data

  ---

  ### Known Production Gotchas

  - `/api/honeytokels` — production endpoint is **intentionally misspelled** ("honeytokels" not "honeytokens"). Do NOT rename it.
  - Render free tier cold-starts: ~30s. ML API has 15s timeout; falls back to `risk_score=9` for honeytoken events.
  - Demo mode: logging in as `admin@gmail.com` bypasses all backends and uses mock data.
  - `VITE_FASTAPI_API_URL` should point to `:8001` (Backend API), not `:8000` (ML API).
